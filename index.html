What is Big O?

Big O notation describes how an algorithm's runtime or memory usage grows as the input size n increases. It measures the worst case ‚Äî how slow can it get?

Lower is always better. An O(1) algorithm runs in the same time whether n=10 or n=1,000,000. An O(n¬≤) algorithm on 1,000 items does 1,000,000 operations.
